Standard Operating Procedures

Complete development guidelines for Donkey Ideas

1. Dashboard Navigation Standards
Purpose: Ensure consistent navigation experience across all sections.
  1. Global company selector must be visible on all pages in the sidebar
  2. Active section must be highlighted in blue (#60a5fa)
  3. Top bar must show breadcrumb navigation and auto-save status
  4. All changes must trigger auto-save within 1 second
  5. Page transitions should be instant (no loading screens for navigation)
2. Financial Data Management
Purpose: Maintain accuracy and consistency in financial calculations.
  1. All financial inputs must auto-calculate dependent fields
  2. Gross Profit = Revenue - COGS (automatic)
  3. Net Profit = Gross Profit - Operating Expenses (automatic)
  4. Profit Margin = (Net Profit / Revenue) × 100 (automatic)
  5. All currency values must display with $ symbol and comma separators
  6. Negative values must display in red (#ef4444)
  7. Auto-save must trigger on any input change
3. Valuation Engine Logic
Purpose: Standardize valuation calculations across all companies.
  1. Revenue Multiple Method:
    • Valuation = ARR × Multiple
    • Multiple based on growth rate:
      • 0-20% growth: 3-5x
      • 20-40% growth: 5-8x
      • 40%+ growth: 8-12x
    • Adjust +1x for profit margin > 30%
    • Adjust +0.5x for churn rate < 2%
  2. AI Score Calculation:
    • Growth Score: Normalize growth rate (0-100%+) to 0-100 points
    • Profitability Score: Normalize margin (0-50%+) to 0-100 points
    • Retention Score: Normalize retention (80-100%) to 0-100 points
    • Market Score: Normalize TAM ($100M-$10B+) to 0-100 points
    • Total Score = Average of 4 scores
4. Team & Permission Management
Purpose: Enforce role-based access control consistently.
  1. Owner Role: Full access to everything including company deletion
  2. Admin Role: All access except company deletion
  3. Member Role: Read/write data, no settings or team management
  4. Viewer Role: Read-only access to all data
  5. Investor Role: Read-only access to selected metrics only
  6. Permission checks must happen on both frontend and backend
  7. Unauthorized actions must show error message and not execute
5. Document Library Workflow
Purpose: Maintain document organization and version control.
  1. All uploads must be scanned for viruses before storage
  2. Max file size: 10MB (configurable)
  3. Supported formats: PDF, DOCX, XLSX, PPTX, JPG, PNG, ZIP
  4. Every file modification creates a new version (v1, v2, v3...)
  5. Version history must track: user, timestamp, changes description
  6. Users can restore any previous version
  7. Files must be stored in S3 or equivalent cloud storage
  8. Generate thumbnails for image files automatically
6. Activity Logging Standards
Purpose: Create comprehensive audit trail for compliance.
  1. Log all user actions: create, read, update, delete
  2. Store: user ID, action type, entity type, entity ID, timestamp, changes (JSON)
  3. Actions to always log:
    • Financial data changes
    • Document uploads/modifications
    • Team member additions/removals
    • Permission changes
    • Investor access grants
    • Company settings changes
  4. Logs must be immutable (cannot be edited or deleted)
  5. Retention: 7 years for compliance
7. AI Integration Guidelines
Purpose: Standardize AI features and API usage.
  1. Deck Generation:
    • Pull data from Business Profile, Financials, Traction
    • Use GPT-4 API for content generation
    • Template-based with AI-enhanced content
    • Allow manual editing of AI-generated content
  2. Financial Forecasting:
    • Analyze last 12 months of data
    • Generate 12-month forward projections
    • Provide confidence scores (0-100%)
    • Offer conservative/realistic/optimistic scenarios
  3. Cost Management:
    • Implement request caching to reduce API calls
    • Set per-user monthly token limits
    • Track costs per company and per user
8. UI/UX Design Standards
Purpose: Maintain consistent visual design across the platform.
  1. Colors:
    • Primary: #60a5fa (blue)
    • Success: #10b981 (green)
    • Warning: #eab308 (yellow)
    • Error: #ef4444 (red)
    • Background: #0A0A0A (black)
  2. Typography:
    • Font: -apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI'
    • Page Title: 32px, weight 700
    • Card Title: 18px, weight 600
    • Body Text: 14px, weight 400
  3. Spacing:
    • Section margin: 2rem
    • Card padding: 1.5rem
    • Grid gap: 1.5rem
  4. Buttons:
    • Primary: Blue background, black text
    • Secondary: Transparent with border
    • Height: 40px (standard), 32px (small)
    • Border radius: 6px
9. Performance Requirements
Purpose: Ensure fast, responsive user experience.
  1. API response time: < 200ms (p95)
  2. Page load time: < 2 seconds (p95)
  3. Database queries: < 100ms (p95)
  4. Auto-save debounce: 1 second
  5. Implement lazy loading for images
  6. Use code splitting for frontend bundles
  7. Cache expensive computations (Redis)
  8. Compress API responses (gzip)
10. Testing & Quality Assurance
Purpose: Maintain code quality and prevent bugs.
  1. Unit Tests: 80%+ code coverage required
  2. Integration Tests: Test all API endpoints
  3. E2E Tests: Critical user flows must be tested
  4. Load Testing: Support 1000 concurrent users
  5. Security Testing: Monthly vulnerability scans
  6. All PRs must pass tests before merge
  7. Manual QA for UI changes
